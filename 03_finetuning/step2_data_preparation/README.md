好的数据 = 好的模型

## 📊 数据格式示例

### 标准格式 (对话格式)

```json
{
  "messages": [
    {"role": "system", "content": "你是一个交通法律专家"},
    {"role": "user", "content": "醉驾会受到什么处罚？"},
    {"role": "assistant", "content": "醉酒驾驶机动车的，由公安机关..."}
  ]
}
```

### 简化格式 (指令格式)

```json
{
  "instruction": "醉驾会受到什么处罚？",
  "output": "醉酒驾驶机动车的，由公安机关..."
}
```

## ✅ 高质量数据的标准

1. **准确性**: 答案必须正确
2. **一致性**: 格式和风格统一
3. **多样性**: 覆盖各种问题类型
4. **完整性**: 答案详细完整
5. **相关性**: 与目标任务相关

## 📏 数据量建议

| 任务复杂度 | 最少样本 | 推荐样本 | 理想样本 |
|------------|---------|---------|---------|
| 简单 (单一格式) | 50 | 200 | 500+ |
| 中等 (多种场景) | 200 | 500 | 2000+ |
| 复杂 (多任务) | 500 | 2000 | 10000+ |
